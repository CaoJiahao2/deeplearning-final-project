## 5. 任务 1：图片描述生成

### 5.1 任务背景

课程提供的图片没有人工标注，因此需要先使用已有的多模态大模型为所有图片生成多维度描述信息。

### 5.2 核心要求

需要完成以下工作：

1. 为训练集和验证集图片设计合适的 prompt。
2. 使用已有的多模态大模型为图片生成描述信息。
3. 通过 prompt engineering 技巧提升描述质量和信息量。
4. 尝试不同 prompt，并比较不同 prompt 生成描述的差异。
5. 总结优秀 prompt 的写作技巧。

### 5.3 可选择的描述方向

生成图片描述时，可以重点关注以下方向，也鼓励探索其他方向：

1. **物体识别**：识别图片中包含的物体。
2. **简短故事生成**：根据图片生成 1-2 句话的简短故事。
3. **图片类别判断**：判断图片所属类别，例如：
   - 表情包
   - 真实照片
   - 网络图片
   - 软件截图
4. **其他方向**：可自行尝试其他维度或技巧，体验不同难度的挑战。

### 5.4 任务 1 的建议输出

为了便于后续训练，建议将自动生成的标注保存为结构化文件，例如：

```text
data/
  train_images/
  val_images/
annotations/
  train_captions.jsonl
  val_captions.jsonl
  prompt_versions/
```

每条标注建议包含：

```json
{
  "image_id": "example.jpg",
  "split": "train",
  "prompt_version": "v1",
  "caption": "...",
  "objects": ["..."],
  "category": "真实照片",
  "short_story": "..."
}
```

## 6. 任务 2：基于深度学习的图片描述自动生成

### 6.1 任务背景

使用任务 1 中由多模态大模型生成的图片描述作为标注，训练一个图像描述生成模型。

### 6.2 核心要求

需要完成以下工作：

1. 使用生成的图片描述作为训练标注。
2. 利用 CNN-RNN 或其他常见图像描述生成模型架构进行训练。
3. 在训练集上训练图像描述模型。
4. 在验证集上进行验证。
5. 尝试不同模型结构，并通过实验讨论不同结构的影响。

### 6.3 模型设计建议

建议在模型设计中加入注意力机制，并通过实验讨论：

- 加入注意力机制前的模型性能。
- 加入注意力机制后的模型性能。
- 注意力机制对生成质量、训练稳定性、指标表现的影响。

可考虑的模型方向包括但不限于：

- CNN encoder + RNN decoder
- CNN encoder + LSTM decoder
- CNN encoder + GRU decoder
- Attention-based image captioning model
- Transformer-based image captioning model
- 其他常见图像描述生成架构

## 7. 模型评估

### 7.1 传统指标

模型性能评估可以使用传统指标，例如：

- BLEU
- METEOR
- CIDEr

### 7.2 其他评估方法

鼓励尝试新的评估方法，例如：

- model-based metrics
- 基于大模型的语义一致性评估
- 人工抽样分析
- 错误类型分析

### 7.3 报告中需要讨论的问题

报告中应讨论：

1. 不同评估指标的优劣。
2. 如何更好地评估生成图片描述的质量。
3. 自动指标与人类感知质量之间可能存在的差异。

## 8. 拓展测试，选做

可以收集自己感兴趣领域的图片作为额外训练集或测试集，体验模型迁移能力。

可选领域包括：

- 抽象画
- 古典艺术画作
- 医学影像
- 其他自选领域图片

需要讨论模型面对不同领域图片时的表现，例如：

- 是否能正确识别领域特征。
- 是否出现明显 domain shift。
- 描述是否准确、完整、自然。
- 模型是否对训练集领域过拟合。

## 11. 编程 agent 可执行任务拆解

### 11.1 数据准备

- 下载 Train 和 Val 数据集。
- 建立统一数据目录。

### 11.2 多模态大模型标注生成

- 设计至少多个 prompt 版本。
- 对所有训练集图片生成描述。
- 对所有验证集图片生成描述。
- 保存每张图片对应的描述结果。
- 比较不同 prompt 的输出差异。
- 总结最终采用的 prompt 策略。

### 11.3 图像描述模型训练

- 构建图像 encoder。
- 构建文本 decoder。
- 构建词表或 tokenizer。
- 使用生成标注训练模型。
- 保存模型 checkpoint。
- 记录训练日志。

### 11.4 注意力机制实验

- 实现无注意力 baseline。
- 实现带注意力版本。
- 使用相同数据和评估流程进行对比。
- 记录并分析性能变化。

### 11.5 验证与评估

- 在验证集上生成 captions。
- 计算 BLEU、METEOR、CIDEr 等指标。
- 可选：加入 model-based metrics。
- 抽样展示模型生成结果。
- 分析失败案例。

### 11.6 报告与文档

- 撰写 Final Report。
- 编写 README。
- 编写环境依赖说明。
- 编写运行、训练、测试和部署说明。
- 可选：完成在线 Demo。
